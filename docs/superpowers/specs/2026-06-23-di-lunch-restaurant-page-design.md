# DI 팀 점심 맛집 페이지 — 설계 문서

작성일: 2026-06-23

## 1. 목적

DI 팀이 점심 메뉴를 정할 때 매번 고민하는 문제를 해결한다. 지금까지 갔던
맛집과 주변 맛집을 한곳에 저장해두고, 그날그날 필터로 후보를 좁혀가며 결정한다.
팀원 누구나 맛집을 등록할 수 있고, 등록 즉시 모두에게 반영된다.

## 2. 기술 구성

- 순수 HTML/CSS/JS (빌드 단계 없음)
- Firebase Firestore (CDN으로 로드, 실시간 동기화)
- GitHub Pages 배포 — 파일을 push하면 바로 반영
- Firestore `onSnapshot` 실시간 리스너로 팀원 등록 내용이 모두에게 즉시 반영

빌드 도구나 프레임워크를 쓰지 않는다. push → 배포 흐름을 최대한 단순하게 유지하는 것이 목표.

## 3. 데이터 모델

Firestore `restaurants` 컬렉션. 문서 1개 = 맛집 1개.

| 필드 | 타입 | 설명 |
|---|---|---|
| `name` | string | 이름 (필수) |
| `category` | string | 한식 / 중식 / 일식 / 양식 / 분식 / 카페 / 단체예약 / 기타 → 필터 |
| `distance` | string | 5분이내 / 10분이내 / 15분이상 → 필터 |
| `price` | string | ~1만원 / 1~1.5만원 / 1.5만원+ → 필터 |
| `memo` | string | 대표메뉴·한줄평 (선택) |
| `createdAt` | timestamp | 등록 시각 (정렬용) |

## 4. 화면 구성 (단일 페이지)

- **상단**: 제목 + "맛집 등록" 버튼
- **필터 바**: 카테고리 / 거리 / 가격대 선택(드롭다운 또는 칩). 여러 조건을 조합해 좁힌다.
  "전체 보기" 초기화 버튼 제공.
- **목록**: 필터된 맛집을 카드 그리드로 표시 (이름·카테고리·거리·가격·메모).
  각 카드에 삭제 버튼. 현재 결과 개수 표시. 결과 0건일 때 안내 문구.
- **등록 모달**: "맛집 등록" 클릭 시 폼 팝업 (이름/카테고리/거리/가격/메모 입력 → 저장).
  이름 미입력 시 저장 차단.

## 5. 동작 흐름

1. 페이지 로드 → Firestore `restaurants`를 `onSnapshot`으로 구독, `createdAt` 내림차순 정렬.
2. 데이터 변경(추가/삭제) 발생 → 리스너가 최신 목록을 받아 현재 필터 조건으로 다시 렌더링.
3. 등록: 모달 폼 제출 → Firestore에 문서 추가 → 리스너가 자동 반영.
4. 삭제: 카드 삭제 버튼 → 확인 후 Firestore 문서 삭제 → 리스너가 자동 반영.
5. 필터: 드롭다운/칩 변경 → 메모리상의 목록을 클라이언트에서 필터링해 다시 렌더링
   (재요청 없음).

## 6. 파일 구조

```
index.html          # 화면 골격
style.css           # 스타일
app.js              # Firestore 연동 + 필터 + 렌더링
firebase-config.js  # Firebase 프로젝트 설정값 (사용자가 채움)
README.md           # Firebase 프로젝트 생성 & GitHub Pages 배포 가이드
```

## 7. 접근 / 보안

- 완전 공개. 주소를 아는 사람은 누구나 보고 등록/삭제 가능.
- Firestore 보안 규칙은 테스트 모드(`read/write` 허용). 점심 맛집 데이터라 민감도 낮음.
- Firebase 프로젝트는 사용자가 직접 생성(무료). README에 단계별 가이드를 넣고,
  발급받은 config 값만 `firebase-config.js`에 채워 넣으면 동작한다.

## 8. 에러 처리

- Firebase config 미설정 시: 화면에 "Firebase 설정이 필요합니다 (README 참고)" 안내 표시.
- Firestore 읽기/쓰기 실패 시: 콘솔 로그 + 사용자에게 간단한 alert/토스트 안내.
- 등록 폼 필수값(이름) 누락 시 저장 차단 및 안내.

## 9. 범위 밖 (YAGNI)

- 로그인/인증, 수정(edit) 기능, 별점·사진·지도 링크, 투표·랜덤뽑기.
- 필요해지면 별도 작업으로 추가한다.
